Core Features
Powered by state-of-the-art deep learning models
👁
Real-time Face Recognition
Uses ArcFace ResNet50 trained on 600K identities to accurately identify registered students even in challenging lighting and angles.
👤
Full Body Detection
YOLOv8s detects persons even when their face is not visible, enabling continuous tracking throughout the camera frame.
🔁
Cross-Camera Tracking
OSNet body re-identification assigns consistent IDs to persons across multiple cameras using appearance embeddings.
⚠
Unknown Person Alerts
Automatically detects and alerts when an unregistered person is detected. Saves snapshots and raises real-time dashboard alerts.
🎯
Detection Zones
Configurable detection zones focus recognition on specific areas like entrances and corridors, reducing false detections.
📊
Live Dashboard
Real-time web dashboard with live camera feeds, detection logs, alerts with snapshots, and cross-camera tracking stats.
Known Limitations
Transparent about current system constraints
🔅
Lighting Dependency
Recognition accuracy drops significantly in very dark environments or when faces are backlit. Adequate lighting is required for reliable detection.
📷
Camera Angle Sensitivity
Top-down mounted cameras reduce face detection accuracy. Best results are achieved with cameras positioned at face level or 45-degree angle.
👁
Face Occlusion
Masks, caps, or hands covering the face prevent recognition. The system falls back to body-based tracking but cannot confirm identity.
👥
Similar Appearance Confusion
Persons with very similar facial features may occasionally be confused. Mitigated by open-set rejection with confidence margin thresholds.
🛠
Hardware Requirement
Requires a dedicated GPU for real-time processing. CPU-only mode is supported but results in significantly reduced frame rates.
🌎
Local Network Only
Currently operates on a local network. Remote access and cloud deployment require additional configuration and infrastructure.
